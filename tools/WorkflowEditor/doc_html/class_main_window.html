bool MainWindow::build_wf_status_msg ( int missing_inputs,
int missing_some_outputs,
int missing_all_outputs,
int disconnected_repos,
QString & status_msg )
private

Build a warning message stating the status of the workflow, including the number of tasks and repositories in the several possible states.

Parameters
[in]missing_inputsTotal number of tasks for which at least one input has not been assigned.
[in]missing_some_outputsTotal number of tasks for which at least one output has not been assigned.
[in]missing_all_outputsTotal number of tasks for which none of their outputs have been assigned.
[in]disconnected_reposTotal number of repositories that have neither incoming nor outcoming connections - that is, that are involved in zero connections, so they are disconnected.
[out]status_msgThe message with the information about the current status of the workflow.
Returns
True if the workflow is complete (that is, all required connections exist), false otherwise.

◆ load_workflow_chain()

void MainWindow::load_workflow_chain ( QString & wf_path)
private

Load a workflow as well as the toolkit on which it relies.

Parameters
[in]wf_pathPath to the xml file containing the definition of the workflow.

The method first tries to load the workflow file using load_workflow(); then, asks for the toolkit on which the launcher relies. If everything is correct, then this method displays the workflow on the screen.

Note that this method uses message boxes to tell the user about the possible errors detected when loading the launcher.
